## 2. 数据库 Schema（SQLite）草案

使用 `sqlx` 管理，迁移时自动创建。

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    student_id TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    role TEXT NOT NULL DEFAULT 'user',   -- 'admin' 或 'user'
    must_change_password INTEGER NOT NULL DEFAULT 1,
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 会话表 (Cookie Token)
CREATE TABLE IF NOT EXISTS sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    token TEXT NOT NULL UNIQUE,
    user_id INTEGER NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    expires_at TEXT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- 打印任务表
CREATE TABLE IF NOT EXISTS print_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    file_name TEXT NOT NULL,               -- 原始文件名
    stored_path TEXT NOT NULL,             -- 上传文件路径
    preview_path TEXT,                     -- 转换后 PDF 路径
    page_count INTEGER NOT NULL DEFAULT 0,
    odd_even TEXT NOT NULL DEFAULT 'all', -- 'all', 'odd', 'even'
    status TEXT NOT NULL DEFAULT 'queued', -- 'queued','printing','done','cancelled','pending_review'
    submitted_at TEXT NOT NULL DEFAULT (datetime('now')),
    completed_at TEXT,
    cancelled_by TEXT,                     -- NULL 或 'user'/'admin'
    review_reason TEXT,                    -- 管理员拒绝理由或审核说明
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- 每日用量记录（用于限额计算和统计）
CREATE TABLE IF NOT EXISTS daily_usage (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    date TEXT NOT NULL,                    -- 'YYYY-MM-DD'
    page_count INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE(user_id, date)
);

-- 全局配置键值对
CREATE TABLE IF NOT EXISTS global_config (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);

-- 审计/操作日志（可选，方便扩展）
CREATE TABLE IF NOT EXISTS audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    action TEXT NOT NULL,
    details TEXT,
    ip TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
);
```

**初始化配置数据：**
```
INSERT OR IGNORE INTO global_config (key, value) VALUES ('daily_limit', '50');
INSERT OR IGNORE INTO global_config (key, value) VALUES ('admin_qq', '');
INSERT OR IGNORE INTO global_config (key, value) VALUES ('admin_student_id', '');
```
管理员账号在首次启动时根据 `config.toml` 中的 `initial_admin_student_id` 自动创建。

## 3. REST API 草案（Base: `http://10.18.47.101:80/api`）

全部返回 JSON。除登录外，所有请求需携带 Cookie 中的 `session_token`。

### 3.1 认证相关

- `POST /api/auth/login`  
  Body: `{ student_id, password }`  
  → 成功返回 `200 { user: { student_id, role, must_change_password } }`，并 Set-Cookie。
- `POST /api/auth/logout`  
  → 清除服务端 session，清除 Cookie。
- `GET /api/auth/me`  
  → 返回当前登录用户信息。
- `POST /api/auth/change-password`  
  Body: `{ old_password, new_password }`  
  → 成功同时清除 must_change_password 标志。

### 3.2 用户功能

- `GET /api/user/quota`  
  → 返回 `{ used_today, limit }`。
- `GET /api/user/history?page=1&per_page=20`  
  → 返回个人打印历史分页。

### 3.3 打印功能
- `POST /api/print/upload`  
  multipart/form-data，字段 `files`（多个文件）。  
  → 接收文件，临时存储，返回文件列表，每个文件含 `temp_id`、原始名、页数（若支持）、预览 URL。
- `GET /api/print/preview/{temp_id}`  
  → 返回转换后的 PDF 文件流（用于前端 pdf.js 渲染）。
- `POST /api/print/submit`  
  Body:  
  
  ```json
  {
    "files": [
      { "temp_id": "...", "odd_even": "all" }
    ]
  }
  ```
  → 检查限额，创建任务（状态 `queued` 或 `pending_review`），返回任务列表。若超额需要二次确认（前端保证），但接口内仍会执行创建并设为待审核。
- `DELETE /api/print/tasks/{task_id}`  
  → 用户取消自己未开始的任务，管理员可取消任意未完成任务。

### 3.4 队列查看
- `GET /api/queue`  
  → 返回所有未完成任务（`queued`、`printing`、`pending_review`），格式：
  ```json
  {
    "tasks": [
      {
        "id": 1,
        "status": "queued",
        "page_count": 3,
        "odd_even": "all",
        "submitted_at": "…",
        "owner_name": "张三",      // 若是管理员或自己，可见学号，否则隐藏
        "file_name_visible": false,
        "file_name": "xxx.pdf"    // 仅管理员或任务所有者看到，否则 null
      }
    ],
    "paused": false
  }
  ```

### 3.5 WebSocket 推送
- `GET /api/ws/queue`  
  → 升级为 WebSocket，服务端在以下事件时推送 JSON 消息：
  - 新任务加入队列
  - 任务状态变化（开始打印、完成、取消、进入待审核等）
  - 队列暂停/继续
  - 打印机错误（若可检测）
  
  客户端连接后无需发送消息，仅接收。

### 3.6 管理员接口（需要 admin 角色）

- `GET /api/admin/users`  
  → 用户列表，支持分页。
- `POST /api/admin/users/import`  
  multipart，上传 Excel（`.xlsx`），仅读第一列学号。返回导入结果（成功/失败列表）。
- `DELETE /api/admin/users/{user_id}`  
  → 删除用户，同时取消其所有未完成任务。
- `POST /api/admin/users/{user_id}/reset-password`  
  Body: `{ new_password }` 或默认重置为学号。
- `GET /api/admin/queue`  
  → 同普通队列但文件名全部可见，可返回额外字段（如 `user_id`）。
- `POST /api/admin/queue/pause`  → 暂停队列，`POST /api/admin/queue/resume`。
- `DELETE /api/admin/tasks/{task_id}`  
  → 管理员取消任务（可带 `reason`）。
- `GET /api/admin/review`  
  → 获取所有 `pending_review` 任务。
- `POST /api/admin/review/{task_id}/approve`  
  → 同意，任务变 `queued`。
- `POST /api/admin/review/{task_id}/reject`  
  Body: `{ reason }` → 任务变 `cancelled`。
- `GET /api/admin/stats`  
  → 返回所有用户总打印页数降序排列：`[{ student_id, total_pages, total_tasks }]`。
- `PUT /api/admin/config`  
  Body: `{ key, value }` → 更新配置（限额、QQ 等）。
- `POST /api/admin/transfer`  
  Body: `{ new_admin_student_id }` → 转让管理员身份。

## 4. 核心服务逻辑说明

### 4.1 文档转换

- 上传文件存入 `uploads/`，根据扩展名判断类型。
- 非 PDF 文件通过 PowerShell 脚本调用 Word/Excel/PPT 另存为 PDF，存到 `previews/`，名称 `{uuid}.pdf`。图片文件用 Rust 库（如 image + printpdf）合成 PDF。
- 转换成功后更新 `preview_path` 和 `page_count`。

### 4.2 打印执行
- 队列管理器是一个后台 task，循环检测 `status='queued'` 的任务，按 `id` 顺序获取。
- 调用命令：`print /d:"HP LaserJet P1106" "preview_path"` 或使用 `ShellExecuteW`。
- 提交后任务状态改为 `printing`，若无法获知真实打印结果，延迟数秒后直接标记 `done`（并累加当日用量）；若可监测，等待打印作业完成再标记。

### 4.3 限额与审核

- 提交时计算 `today = date, used + page_count` 是否超过限额。
- 若超过：任务状态 `pending_review`，不计数。
- 管理员同意后：更新状态为 `queued`，**此时检查限额**，若仍超限则拒绝（或再次待审核），批准时允许超出（即不更新用量，但要留记录）。建议：同意即允许打印，并在用量中计入超出部分（实际张数），不阻断；或者简单处理：同意时不再检查限额，直接入队，并增加当日用量（可能使数值超过限额），方便统计。根据需求“管理员可同意或拒绝，同意后进入队列”，我们执行后者，并在统计时以实际打印为准。

### 4.4 定时任务
- 使用 `tokio::time::interval` 实现：
  - 每 10 秒检查队列是否有新任务可打印。
  - 每日零点触发重置：新建日期记录（`daily_usage` 按日期查询自然隔离），无显式重置动作。

### 4.5 WebSocket 推送

- 维护一个连接集合，当任务状态改变时（由 `queue_manager` 触发），向所有连接的客户端广播更新。事件类型可定义枚举，序列化后发送。

## 5. 前端交互初步设想（Vue 3）

- 使用 Vue Router 控制页面，路由守卫根据角色显示不同导航。
- 页面组件树：
  - `LoginView.vue`
  - `MainLayout.vue`（侧边栏或顶部导航）
    - `Dashboard.vue`（今日用量）
    - `SubmitPrint.vue`（拖拽上传区域、文件列表、预览弹窗、单双面选择、提交确认）
    - `QueueView.vue`（卡片队列，自动更新通过 WebSocket 或轮询）
    - `MyHistory.vue`（分页历史）
    - `Settings.vue`（修改密码）
    - （管理员）`UserManagement.vue`
    - （管理员）`AdminQueue.vue`（含暂停/继续/取消）
    - （管理员）`ReviewCenter.vue`
    - （管理员）`StatsView.vue`
    - （管理员）`SystemSettings.vue`

- 状态管理简单可用 Vue 的 `reactive` + `provide/inject`，无需引入 Pinia 但可以使用以便扩展。
- 使用 `axios` 封装请求，自动附带 Cookie（`withCredentials: true`）。
- 预览使用 `vue-pdf-embed` 或直接嵌入 `<iframe>` 指向 `/api/print/preview/...`。
